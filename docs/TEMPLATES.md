## üß≠ Overview

Every call to buildstamp generates metadata in one of the supported formats.
You can select a format using the CLI flag --format or the format key in your config file.
```
buildstamp --format json --out ./Builds/buildinfo.json

```
#### Supported formats:

|Format	|Extension	|Description|
|----|--------|----|
|json	|.json	|Machine-readable metadata|
|text	|.txt	|Human-readable text output|
|md	|.md	|Markdown for release notes|
|cs	|.cs	|C# static class for embedding|
|custom	|any	|User-defined template (Pro)|

## üß© Built-In Formats

Each built-in format uses a default template internally.
You can override or extend them in the Pro edition by providing a .template file.

## üß† Template Variables

The following placeholders are available in all formats:

|Variable	|Description|
|----------|---------|
|{{ project }}	|Project name|
|{{ version }}	|Version number|
|{{ branch }}	|Git branch name|
|{{ commit }}	|Git commit hash|
|{{ timestamp }}	|UTC timestamp|
|{{ tags }}	|Comma-separated list of tags|
|{{ isoDate }}	|ISO date (YYYY-MM-DD)|
|{{ isoTime }}	|ISO time (HH:MM:SS)|
|{{ hashShort }}	|Short commit hash|
|{{ env.KEY }}	|Environment variable (env.BUILD_NUMBER)|

## üß∞ Custom Templates (Pro)

Custom templates are a Pro-only feature, allowing full control over output layout.

#### Example: build-template.txt
```
Project {{ project }} (v{{ version }})
Commit: {{ commit }} on branch {{ branch }}
Built:  {{ isoDate }} {{ isoTime }} UTC
Tags:   {{ tags }}

```
#### Usage
```
buildstamp --template ./templates/build-template.txt --out ./Builds/custom.txt

```
##### Example: build-template.json
```
{
  "meta": {
    "name": "{{ project }}",
    "version": "{{ version }}"
  },
  "build": {
    "timestamp": "{{ timestamp }}",
    "commit": "{{ hashShort }}"
  }
}

```
This allows you to define JSON with any schema you want, compatible with downstream tools.

## üß© Template Syntax Reference

BuildStamp templates use double-brace placeholders ({{ key }}) for variable substitution.
Whitespace, indentation, and line breaks are preserved exactly.

|Expression	|Result|
|-------|-------|
|{{ project }}	|Project name|
|`{{ tags	|join(", ") }}`|
|{{ env.BUILD_NUMBER }}	|Environment variable|
|`{{ commit	|upper }}`|
|`{{ timestamp	|date:"YYYY-MM-DD HH:mm:ss" }}`|

## üóÇÔ∏è Naming & Path Conventions

Default output naming uses the file extension of the selected format.

|Format	|Default Output	|Example|
|----|--------|----|
|JSON	|buildinfo.json	|/Builds/buildinfo.json|
|Markdown	|BUILD_INFO.md	|/Builds/BUILD_INFO.md|
|C#	|BuildInfo.cs	|/src/BuildInfo.cs|
|Text	|buildinfo.txt	|/Builds/buildinfo.txt|

You can override it via --out or out in config.
